## Documentation Guidelines

Technical documentation should:

1. **Be Precise**: Use exact commands, file paths, and code examples
2. **Be Reproducible**: Readers should be able to follow along successfully
3. **Include Examples**: Show concrete examples of usage
4. **Note Prerequisites**: Clearly state what's needed before starting
5. **Keep Updated**: Update when implementation changes
